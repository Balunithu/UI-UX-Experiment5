# UI-UX-Experiment5

## Aim:
To analyze user experience problems in two different digital systems (a Health & Wellness App and a Local Harvest E-commerce platform), identify user pain points, and propose UX insights using problem statements, research methods, and user personas through an interactive tab-based interface.

## Algorithm:
Step 1: Start the program and open the UX project webpage.

Step 2: Create a navigation bar with three tabs – Home, Scenario 1, and Scenario 2.

Step 3: Design the Home tab to display the project overview and GitHub repository link.

Step 4: Create Scenario 1 interface representing the Health & Wellness App dashboard.

Step 5: Display daily wellness tracking goals such as exercise, sleep, water intake, meditation, and meals.

Step 6: Show a generic wellness tip to demonstrate the problem of impersonal suggestions.

Step 7: Identify and display the UX pain point where users feel overwhelmed by too many tracking tasks.

Step 8: Create Scenario 2 interface representing a Local Harvest E-commerce shopping cart.

Step 9: Display a product item with price and substitution policy information.

Step 10: Provide a preference input field for alternative product selection.

Step 11: Highlight the UX issue where users cannot properly control substitution preferences.

Step 12: Display the diagnosis explaining the user friction caused by unclear substitution policies.

Step 13: Allow users to switch between scenarios using tab navigation.

Step 14: End the program.

## Result:
Thus, the UX analysis for two different scenarios was successfully implemented using a tab-based interface. The system demonstrates how poor personalization and lack of user control can create usability problems in digital products. The interface clearly highlights the user pain points and helps in understanding UX issues in health applications and e-commerce platforms.
